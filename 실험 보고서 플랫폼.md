# 심층순환 실험 보고서 플랫폼 — 정리 (v2)

## 무엇을 만들었나

학생이 브라우저에서 실험 보고서를 작성하고, 완성본을 **PDF로 저장해 직접 제출**하는 웹페이지.
서버가 없다. HTML 파일 하나(`deep_circulation_report.html`)로 전부 동작한다.

이번 버전은 "이 플랫폼 자체를 여러 실험에 재사용할 수 있는 하나의 폼으로 정립한다"는 목표로,
디자인을 흑백으로 정리하고, 개념 학습 부분을 크게 보강했다.

---

## 최종 구조

### 탭 6개

| # | 탭 | 내용 |
|---|---|---|
| 01 | 정보 입력 | 반 · 팀명 · 학번 · 이름 · 역할 |
| 02 | 실험 소개 | 두 실험 설명 + 내용 확인 체크 + 실험실 안전 수칙 + 확인 체크 |
| 03 | 개념 | 밀도 결정 요인(빈칸) → T-S도 읽어보기(연습) → 밀도류(빈칸) → 심층 순환(빈칸) |
| 04 | 실험 | 수온·염분 입력 + T-S 다이어그램 + 사진 2장 |
| 05 | 결론 및 고찰 | 해석 / 결론 / 기타 |
| 06 | 보고서 | 미리보기 + PDF 저장 + 제출 링크 |

앞 5개 탭은 다 채워야 다음으로 넘어간다(체크박스 포함). 빠진 게 있으면 그 탭으로 데려가고 빈 칸을 붉게 표시한다.

### 핵심 기능

- **T-S 다이어그램** — 수온·염분을 입력하면 등밀도선 위에 점이 실시간으로 찍힌다. 색을 전혀 쓰지 않고 **도형(실험1=원, 실험2=사각형)** 과 **속 빈 점/속 찬 점**으로 네 시료를 구분한다.
- **T-S도 읽어보기 (연습)** — 고정된 두 점 A·B가 표시된 정적 T-S도를 보여주고, 학생이 그 지점의 수온·염분을 직접 읽어 적어보는 연습. 두 점은 **눈금 교점 위에 정확히 위치**하도록 좌표(A: 염분 32·수온 4 / B: 염분 28·수온 16)를 잡았고, 점에서 두 축까지 **점선 보조선**을 내려 값을 바로 따라 읽을 수 있게 했다. 입력 후 **"정답 확인하기"** 버튼을 누르면 실제 값과 비교해 오차 ±1 이내면 체크 표시가 붙는다. 이 연습 결과는 최종 보고서(PDF)에는 포함하지 않는다 — 순수 연습용.
- **밀도 자동 계산** — 해수 상태방정식(UNESCO / EOS-80, 1기압)으로 계산. **단위는 g/cm³** (kg/m³ 결과값을 1000으로 나눠 표시, 소수 3자리).
- **자동 저장** — 입력 즉시 브라우저에 저장. 사진·체크박스 포함. 탭을 닫았다 열어도 이어서 쓴다. 하단 `초기화` 버튼으로 삭제.
- **사진 처리** — EXIF 회전 보정(폰 사진이 눕지 않음), 1100px로 축소.
- **PDF 저장** — 개념 답안(1·2·3 문단으로 분류된 이어 쓰는 글) · 측정값 표 · T-S 그래프 · 사진 2장 · 해석 · 결론 · 기타를 A4 한 문서로. 파일명은 `심층순환_실험보고서_A반_팀명_20100_홍길동.pdf`.

## 선생님이 할 일

### 제출 링크 걸기

주소 뒤에 `?link=` 를 붙여서 학생에게 준다. **파일을 고칠 필요가 없다.**

```
.../deep_circulation_report.html?link=https://classroom.google.com/c/AAA/a/BBB
```

06번 탭에 "저장한 PDF 제출하러 가기" 버튼이 이 주소로 연결된다.
매번 같은 링크라면 파일 안 `DEFAULT_SUBMIT_URL` 에 넣어 두어도 된다.

### 교과서 참고 쪽수 바꾸기

03 개념 탭 안내문("교과서 ○○쪽을 참고하며...")에 쓰이는 쪽수는 파일 상단
`TEXTBOOK_PAGES` 값 하나만 바꾸면 전체에 반영된다. **다른 실험에 이 폼을 재사용할 때 가장 먼저 손댈 곳.**

```js
const TEXTBOOK_PAGES = "24~25";
```

### 배포

학교 홈페이지나 구글 사이트에 파일을 올린다. 그게 전부다.

### 반 표기 방식

드롭다운이 아니라 `[__]반` 형태의 자유 입력이다. 학생이 알파벳 한 글자(대문자 자동 변환, A~Z만 허용)를 입력하면
"A반"처럼 표시된다. 학기마다 반 편성이 달라져도 코드를 고칠 필요가 없다.

### 팀명

드롭다운이 아니라 자유 입력이다. 모둠 번호 대신 팀이 스스로 정한 이름을 쓴다.

## 개념 빈칸 (정답)

| # | id | 문장 | 정답 | 힌트 |
|---|---|---|---|---|
| 1 | b8 | 수온이 ○ | 낮을수록 | 낮을수록 / 높을수록 |

| 2 | b1 | 밀도가 ○ 해수는 아래쪽으로 | 큰 | 큰 / 작은 |
| 2 | b2 | 밀도가 ○ 해수는 위쪽으로 | 작은 | 큰 / 작은 |
| 2 | b3 | 이러한 해류를 ○라고 한다 | 밀도류 | ㅁㄷㄹ |
| 3 | b4 | 결빙이 일어나면 염분이 ○ | 높아져 | 높아져 / 낮아져 |
| 3 | b5 | 밀도가 ○ | 커진다 | 커진다 / 작아진다 |

## 디자인

- **서체** — Pretendard 하나로 완전히 통일. 이전에는 숫자·탭 번호에 JetBrains Mono를 따로 썼지만, 지금은 `--mono` 변수가 `--sans`(Pretendard)를 그대로 가리키게 해서 폰트가 전부 같다.
- **크기** — 기본 17px, 웬만한 텍스트는 최소 11pt(≈14.7px) 이상. 라벨·힌트·탭·표처럼 작았던 요소들을 전반적으로 키웠고, T-S 그래프 안 글씨(축·범례·등밀도선 숫자)도 함께 키웠다.
- **색 — 완전 흑백** — 처음엔 보라·핑크·민트 조합의 화려한("MZ풍") 테마로 바꿔봤지만 "너무 튄다"는 피드백으로 전부 되돌렸다. 지금은 잉크색(거의 검정) 하나와 회색 몇 단계, 그리고 오류 표시용 빨강 하나만 쓴다.
- **T-S도 데이터 구분** — 색 대신 **도형**을 쓴다. 실험 1은 원, 실험 2는 사각형. 같은 실험의 두 시료는 속 빈 점 ○ / 속 찬 점 ● 으로 구분한다. 범례·표의 마커·그래프의 점이 모두 이 규칙으로 통일되어 있다.
- 그림자 없음. 그라디언트 없음. 이모지 없음(중간에 넣어봤다가 "너무 튄다"는 이유로 뺐다).

## 버려진 방향과 이유

기록으로 남긴다. 다시 그쪽으로 갈 이유가 없다는 뜻이다.

### 화려한 색 테마 ("MZ풍") (폐기)

보라·핑크·민트 그라디언트 + 이모지로 바꿔봤지만, 학생용 표준 폼으로 여러 실험에 재사용하기엔
과하다는 판단으로 흑백으로 되돌렸다. 대신 T-S도의 데이터 구분은 색이 아니라 도형(원/사각형)으로 해결했다.

### 개념 빈칸을 없애고 읽기 자료로 전환 (폐기)

"빈칸이 너무 많은 답을 요구할 것 같다"는 우려로 한 번 빈칸을 다 없애고 완성된 문장(읽기 자료)으로
바꿔봤지만, "빈칸은 있어야 한다"는 피드백으로 즉시 복원했다. 대신 "교과서 ○○쪽을 참고하며
채우라"는 안내문으로 부담을 낮추는 절충을 택했다.

### Google Sheet + Apps Script (폐기)

처음엔 사진을 드라이브에 올리고 시트에 기록하는 구조였다. **제출을 클래스룸으로 받는다면 이 전부가 불필요하다.** 배포·권한·동시접속·HEIC 업로드 실패 같은 실패 지점이 통째로 사라진다.

**대신 잃은 것**: 개념 빈칸 자동 채점, 측정값이 한 표에 모이는 것. 학생 수가 많아 이게 아쉬워지면, PDF는 그대로 두고 채점용 요약 데이터만 시트로 따로 보내는 절충이 가능하다.

### 모둠 번호 드롭다운 (폐기)

처음엔 모둠 번호 드롭다운이었다가, 반(class)과 함께 자유 입력(팀명)으로 바뀌었다. 학기마다
바뀌는 값을 코드 수정 없이 학생이 직접 쓰게 하는 방향으로 정리했다.

### 보기(word bank) 클릭 (폐기)

빈칸 힌트를 낱말 상자에서 고르게 했다가 되돌렸다. **떠올리기(recall)가 알아보기(recognition)로 바뀌기 때문이다.** 앞의 것들을 채우면 남은 후보가 줄어서 마지막 빈칸은 안 읽고도 찍힌다. 기억에 남기려면 빈 종이에서 꺼내 봐야 한다.

### σt 표기 / 단위 kg/m³ (폐기)

지구과학1에서 안 배우는 σt 기호 대신 실제 밀도 값을 쓴다는 원칙은 유지하되, 단위는
**kg/m³ → g/cm³** 로 바꿨다(라벨뿐 아니라 실제 표시값도 1000으로 나눔). 교과서·모의고사에서
익숙한 단위로 맞춘 것.

### 모눈종이 배경 (폐기)

페이지 전체에 격자를 깔았더니 정보가 아니라 **벽지**가 됐다. 모눈은 눈금을 읽는 기능이 있는 **그래프 안에만** 남겼다. (T-S도 읽어보기 연습에서는 오히려 이 격자를 눈금과 정확히 맞춰 "읽기 보조선"으로 적극 활용했다.)

### 한글 명조 제목 (폐기)

화면에서 한글 명조는 아무리 잘 써도 인쇄물 흉내처럼 보인다.

## 알아 둘 점

- **Pretendard는 CDN(`cdn.jsdelivr.net`)에서 불러온다.** 학교 망에서 막혀 있으면 시스템 기본 고딕으로 떨어진다.
- **아이폰 HEIC 사진**은 PC 브라우저에서 못 읽는다. 페이지에 안내를 넣어 뒀다: 설정 > 카메라 > 포맷 > 높은 호환성.
- **자동 저장은 브라우저별·기기별**이다. 학교 컴퓨터에서 쓰다가 집에서 이어 쓸 순 없다.
- **실험 2에서 소금물이 아닌 것**(설탕물, 색소만)을 쓴다면 측정값이 실제 psu가 아니다. 그 경우 "염분" 라벨을 "용액 농도"로 바꾸는 게 맞다.
- **개념 빈칸·T-S도 읽어보기 모두 자동 채점은 하지 않는다.** T-S도 읽어보기의 "정답 확인하기"는 학생이 스스로 맞는지 비교해보는 자기 점검용이고, 서버로 전송되거나 교사에게 집계되지 않는다.
